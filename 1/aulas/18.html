Explicações importantes

Porque tudo esta sendo dividido em componentes?

Isso é o poder do react, tudo hje em dia em programação é quebrado por componentização, quando menor for o componente será o viavel para testes e exibições em tela.

Pois os testes podem ser feitos com componentes separadamente, facilitando para encontrar erros ou finalizar o processo.

Diferenças entre trabalhar com props e estado?

STATE

STATE é o que controla o que será renderizado dos elementos, se ele mudar tudo sera renderizado novamente(atualizando) todos os componentes, A diferença em ter o state é que você pode manipula-lo para baixo, exemplo disso é q ele fica dentro da home, no inicio do projeto.(ele é um estado global), ele poderá ser usado em todo lugares como props estes componentes q sao filhos poderam receber a propriedades poderá fazeer as mudanças.

Lembrando que todos os componentes usados do state e a props são privadas do componente q estará sendo usada. mais sempre poderá ser passado adiante

O estado salvará na memoria tudo q for passado para ele e assim q o state mudar o react vai re-enderizar(atualizará) a pagina novamente

PROPS

Já a PROPS esta sendo recebida mudará diacordo o estado, mas se mudar a props diretamente o estado nao tem nada haver com isso, sendo que a PROP é algo q se passa para qualquer componente que você quiser.

O react tem um sentido de dados sendo q ele nao retorna para trás, exemplo nada pode estar acima do componente pai, exemplo usado o home, então nunca o estado influenciara em componentes pai
